Interactive proofs with imperfect randomness.¶

The class of languages which admit deterministic interactive proofs is NP. The class of languages (\(\mathrm{IP}\)) which admit probabilistic interactive proofs is PSPACE.

\(\bold{Question:}\) Is \(\mathrm{IP{-}weak}=\mathrm{IP}\)? Can we simulate probabilistic interactive proofs using imperfect random sources?

Dodis et al. :raw-latex:`\cite{DOPS04}`(`pdf <https://cs.nyu.edu/~dodis/ps/1-source.pdf>`__) showed that block sources are sufficient to similate interactive proofs.

Consider the case of (information-theoretic) private-key encryption where parties wish to encrypt a \(b\)-bit value using shared secret key sampled from an imperfect random source \(X\) over \(n\) bis. Bosley and Dodis :raw-latex:`\cite{BD07}`(`pdf <https://www.iacr.org/archive/tcc2007/43920001/43920001.pdf>`__) showed that if such scheme is secure, then one can deterministically extract \(b-\log{n}\) bits from \(X\). Hence, to a large extent, true randomness is inherent for encryption.

Separation between encryption and extraction.¶

They conjecture that extracting \(b\) bits from \(X\) is impossible.

For any extractor \(\mathrm{Ext}:\{0,1\}^n\times\big(\{0,1\}^{\mathrm{poly}(n)}\big)^B\rightarrow\{0,1\}\), there exists distribution \(X\) over \(\{0,1\}^n\) and \(\mathrm{Enc}:\{0,1\}^n\times[B]\rightarrow \{0,1\}^{\mathrm{poly}(n)}\) such that

  • for any \(m_0\neq m_1\), \(\Delta(\mathrm{Enc}(X,m_0),\mathrm{Enc}(X,m_1))=0,\)
  • \(\Delta(\mathrm{Ext}(X, \mathrm{Enc}(X,1),\dots, \mathrm{Enc}(X,B)),U_1)>\Omega(1)\)

where \(U_1\) is the uniform distribution over \(\{0,1\}\), \(B=2^b\) and \(\Delta\) stands for statistical distance.

Bosley and Dodis :raw-latex:`\cite{BD07}` showed that above conjecture is true for \(b\leq \log{n}-\log{\log{n}}\).

Is true randomness inherent for sharing schemes?¶

A randomize function \(\mathrm{share}(m, X)\rightarrow(L,R)\) (which takes a message \(m\) over \(b\) bits as input and uses \(X\) as the random source) is a \(2\)-out-of-\(2\) secret sharing scheme if

  • (Reconstruction) there exists an algorithm \(\mathrm{Rec}\) such that

    \[\text{for every $m$, $\Pr[\mathrm{Rec}(L,R)=m]=1,$}\]

  • (Privacy) for any \(m'\neq m\), \(\Delta(L(m),L(m'))=0\) and \(\Delta(R(m),R(m'))=0\).

\(\bold{Question\colon}\) If \(\mathrm{share}(m,X)\) is a \(2\)-out-of-\(2\) secret sharing scheme, can we deterministically extract random bits from \(X\)?

More background in secret sharing can be found in the survey by Beimel :raw-latex:`\cite{Beimel11}` (pdf).

Beating RT-bound using computational extractor.¶

Radhakrishnan and Ta-Shma
:raw-latex:`\cite{RT97}`(`pdf <https://pdfs.semanticscholar.org/0385/3893e657cb69e51b603d3f1cc1d5c47be5a5.pdf>`__) showed that any seeded extractor with error \(\eps\) suffers from \(2\log{1/\eps}\) entropy loss (entropy loss is the amount of entropy in source and seed substracting output length). Motivated by bypassing this limitation, one approach is to consider computational extractor, whose output is only required to be computationally indistinguishable from uniformly random.
Dachman-Soled et al. :raw-latex:`\cite{DGKM12}` (pdf), together with the result of Dodis et al. :raw-latex:`\cite{DPW14}`(`pdf <https://eprint.iacr.org/2013/708.pdf>`__) showed that any efficient computational extractor beating RT-bound implies one-way function.
\(\bold{Question\colon}\) Can we construct an efficient computational extractor beating RT-bound based on one-way functions?

Krawczyk :raw-latex:`\cite{Kra10}`(`pdf <https://eprint.iacr.org/2010/264.pdf>`__) used extract-then-expand approach and showed a computational extractor for medium-to-high entropy sources. More background and other approaches for constructing compuational extractors can be found in Yevgeniy’s slides and lecture note.
